# G8 — Markup Grouping (S2 group 8)

**Branch:** `feat/g8-grouping` (off `main` @ `b29d0d9`, which carries S2a+G1–G7).
**Plan author:** redline instance, 2026-06-16. **Method:** TDD, subagent-driven at group granularity.

## Goal (from S2 impl plan G8 cut-line)
Add a `group_id` to the markup model so multiple markups can be grouped, move together, and
survive save→reopen. Exit criteria:
1. **Group / ungroup** a multi-selection (one undo frame each).
2. **Group moves together** — selecting one member selects the whole group; dragging moves all.
3. **`/RLGroup` survives save→reopen**; external viewers ignore the unknown key gracefully.

## Precision/architecture invariants (do not violate)
- §5: all geometry math in PDF user space at f64; never read the raster. (Grouping adds no geometry math beyond reusing existing move.)
- Svelte store is in-session SoT; Rust store is a mirror + save buffer. Group/ungroup commits through `store.applyBatch` (ONE undo frame, N mirror ops) — same path as G6/G7 multi-select edits.
- Pure helpers stay DOM/Svelte/clock/UUID-free; `by`/`now`/`groupId` are injected (mirrors `patchAppearance`/`bumpAudit`).
- `id` is immutable; `group_id` is mutable metadata (like `subject`/`layer`).

## Tasks (TDD — failing test first for each)

### T1 — Rust model: `group_id` field (`src-tauri/src/markup/mod.rs`)
- Add `pub group_id: Option<Uuid>` to `struct Markup` (place after `layer`/`contents` metadata; before `audit` is fine — keep struct-literal sites in sync).
- Annotate with `#[serde(default)]` so pre-G8 JSON (no field) deserializes to `None`.
- `Markup::new(...)`: initialize `group_id: None`.
- Update the `from_annotation_dict` struct literal (annotation.rs) — see T2.
- **Tests** (in `mod.rs` tests):
  - `new_markup_has_no_group`: a fresh markup has `group_id == None`.
  - `serde_round_trip_preserves_group_id`: set `group_id = Some(Uuid::new_v4())`, JSON round-trip, assert equal.
  - `serde_default_group_id_when_absent`: deserialize a JSON object **without** `group_id` (e.g. take a serialized markup, strip the key, or build minimal JSON) → `group_id == None`. (Confirms `#[serde(default)]`.)

### T2 — Rust serde: `/RLGroup` round-trip (`src-tauri/src/markup/annotation.rs`)
- `to_annotation_dict`: after the `/RLLayer` block, add
  `if let Some(gid) = self.group_id { d.set("RLGroup", Object::string_literal(gid.to_string())); }`
  (omit the key entirely when `None` — never write an empty/placeholder).
- `from_annotation_dict`: in the struct literal add
  `group_id: get_string(d, b"RLGroup").and_then(|s| uuid::Uuid::parse_str(&s).ok()),`
  (malformed/absent → `None`; foreign annotations have no `/RLGroup` → `None`).
- **Tests** (annotation.rs tests):
  - `grouped_markup_rl_group_round_trips`: build a markup, set `group_id = Some(g)`, `to_annotation_dict` → assert `/RLGroup` string == g; `from_annotation_dict` of that dict → `group_id == Some(g)`. Reuse the existing `assert_roundtrip` helper if it goes through the dict path; otherwise assert explicitly.
  - `ungrouped_markup_omits_rl_group`: `group_id == None` → dict has **no** `RLGroup` key; `from_annotation_dict` → `None`.
  - `foreign_annotation_without_rl_group_is_ungrouped`: a bare foreign dict (no `/RLGroup`) imports with `group_id == None`.

### T3 — TS model: `group_id` on `Markup` (`src/lib/ipc.ts`)
- Add `group_id: string | null;` to `interface Markup` (after `contents`).
- No behavior; this unblocks the TS compile for the helpers/tests below. Any test fixtures that build a `Markup` literal must include `group_id: null`.

### T4 — Pure helper: `patchGroup` (`src/lib/markup-properties.ts` + `.test.ts`)
- Add `export function patchGroup(m: Markup, groupId: string | null, by: UserRef, now: string): Markup`
  returning `bumpAudit({ ...m, group_id: groupId }, by, now)`. Pure; no input mutation.
- **Tests** (markup-properties.test.ts):
  - sets `group_id` to a value and bumps audit (revision+1, modified_by/at).
  - clears `group_id` to `null` when passed `null`.
  - does not mutate the input markup.

### T5 — Pure helper: `expandSelectionToGroups` (`src/lib/markup-select.ts` + `.test.ts`)
- Add `export function expandSelectionToGroups(markups: Markup[], ids: Set<string>): Set<string>`:
  for every selected id whose markup has a non-null `group_id`, add **all** markups sharing that
  `group_id`. Markups with `group_id == null` contribute only themselves. Returns a new Set
  (never mutates `ids`). Ungrouped ids pass through unchanged.
- **Tests** (markup-select.test.ts):
  - selecting one member of a 3-member group returns all 3 ids.
  - selecting an ungrouped markup returns just that id.
  - mixed selection (one grouped + one ungrouped) returns the full group + the loner.
  - empty input → empty set; does not mutate the passed-in set.

### T6 — Viewport wiring (`src/components/Viewport.svelte`)
- **Group-aware selection** — in the SELECT-tool hit branch:
  - Plain click on a hit: `store.selectedIds = expandSelectionToGroups(store.markups, new Set([hit]))` (instead of `new Set([hit])`), THEN set up move with `moveOrigins = store.selectedMarkups.filter((m) => m.page === pageIndex)` (unchanged — it already reads the expanded selection).
  - Shift-click toggle: toggle the hit's **whole group**. Compute `grp = expandSelectionToGroups(store.markups, new Set([hit]))`; if `hit` is already selected, remove all of `grp`; else add all of `grp`.
  - Keep the "select if not already, then move" semantics: if the clicked member's group is already fully selected, just start the move.
- **Keyboard group/ungroup** — in `onKeydown` (alongside Delete/Backspace), when `isSelectTool` and not in the text editor:
  - `(e.metaKey || e.ctrlKey) && e.key === "g"` (no shift): group. Targets = `store.selectedMarkups`. Require **≥2** targets (a 1-markup group is a no-op). `const gid = crypto.randomUUID(); const now = new Date().toISOString(); const pairs = targets.map((m) => ({ before: m, after: patchGroup(m, gid, identity!, now) })); store.applyBatch(pairs);` Keep selection. `e.preventDefault()`.
  - `(e.metaKey || e.ctrlKey) && e.shiftKey && (e.key === "g" || e.key === "G")`: ungroup. Targets = selected markups with `group_id !== null`; clear via `patchGroup(m, null, identity!, now)`; `applyBatch`. `e.preventDefault()`.
  - Guard on `identity` being loaded (same as create tools). Match the existing key-handling order so the zoom `Cmd+0` / multi-click `Enter`/`Escape` branches are untouched.
- Move already moves all `selectedMarkups` → grouped move falls out. No new geometry math.

### T7 — Interaction tests (`src/components/` — mirror G6/G7 interaction tests)
Mount the real Viewport with the mock-IPC store (existing harness/pattern). Script:
- **group**: seed 3 markups (2 to be grouped), select 2, dispatch Cmd+G (metaKey) → assert both share a non-null equal `group_id`, the third is still `null`, and it is ONE undo frame (one `store.undo()` reverts both, group_id back to null).
- **group-aware select**: with a 2-member group, click one member → assert `store.selectedIds` contains both.
- **group move glued**: select via clicking a group member, drag-move → assert both markups' geometry translated by the same delta (reuse the §5 glued assertion style).
- **ungroup**: with a group selected, dispatch Cmd+Shift+G → assert both `group_id == null`, one undo frame.
- Keep assertions on store + SVG where the G6/G7 tests do; no per-op manual GUI.
